Enterprise WAN anomaly detection remains a critical challenge: conventional
rule-based threshold monitors fail to capture slow, multi-metric degradation
patterns that precede service-level agreement breaches.
\citet{schummer2024} propose a supervised machine
learning framework addressing this gap, with two central claims---that
non-linear models outperform a linear baseline, and that feature-engineering
quality dominates model complexity.
To verify these claims at minimal scale, a synthetic 80{,}640-sample
multivariate WAN telemetry dataset is constructed with a realistic 16.4\%
anomaly rate, and four classifiers are evaluated under a time-aware 70/30
train/test split: Logistic Regression (linear baseline), Random Forest,
SVM with RBF kernel, and a PyTorch MLP introduced as a deep-learning extension.
All four models achieve Precision-Recall AUC (\prauc) of approximately
0.310---roughly $2\times$ the random-classifier baseline of 0.164---confirming
that temporal feature engineering carries genuine predictive signal.
The spread across all classifiers is within 0.003 \prauc regardless of model
complexity, directly corroborating the central claim that the feature
representation, not the model family, is the binding performance constraint.
A notable operational finding emerges: the MLP attains 35.6\% recall versus
13.8\% for RF/SVM at the default decision threshold, a difference attributable
to positive-class weighting in the binary cross-entropy loss and with direct
implications for production deployment where missed anomalies carry higher cost
than false alarms.
Complete, self-contained code is provided, enabling full reproduction of all
reported results in under ten minutes on a single CPU.

The detection of performance anomalies in enterprise wide area networks (WANs) is critical 
for service providers and large organizations to uphold strict service-level agreements (SLAs) 
and maintain reliable network operations. Delays in identifying and addressing these anomalies 
may result in SLA violations, diminished user experience, and significant operational 
disruptions. \citep{schummer2024} introduce a supervised machine learning framework to 
address this challenge, advancing two primary claims: non-linear models outperform a linear 
baseline, and the quality of feature engineering has a greater impact than model complexity. 
Due to the proprietary nature of the original dataset, a synthetic yet statistically 
representative multivariate WAN telemetry dataset comprising 80{,}640 samples and a 16.0% anomaly 
rate is constructed to facilitate claim reproduction and evaluation. Temporal feature 
engineering, including rolling statistics, lag features, and multi-horizon aggregation windows, 
is applied within a time-aware 70/30 train/test split. Four supervised classifiers are assessed: 
Logistic Regression (serving as the linear baseline), Random Forest, Support Vector Machine (SVM) 
with a radial basis function (RBF) kernel, and a PyTorch-based multilayer perceptron (MLP) as a 
deep learning extension. Model performance is evaluated using Precision-Recall Area Under the 
Curve (PR-AUC), which is appropriate for imbalanced anomaly detection tasks. All four models 
achieve PR-AUC values near 0.865, approximately 5.4 times higher than the random-classifier 
baseline of 0.160, confirming that temporal feature engineering provides a strong predictive 
signal. The classifiers' PR-AUC scores are within 0.013 of each other, supporting the claim that 
feature engineering quality dominates model complexity. Notably, Logistic Regression attains the 
highest PR-AUC (0.870), marginally surpassing Random Forest (0.868) and SVM (0.865), which 
partially contradicts the non-linearity claim and suggests that the advantage of non-linear 
models may depend on the complexity of real-world telemetry. The complete, reproducible code 
enables full replication in under ten minutes on a single machine.

\section{Introduction}
\label{sec:intro}
% ─────────────────────────────────────────────────────────────────────────────
\begin{comment}
Enterprise Wide Area Networks (WANs) carry latency-sensitive traffic including
financial transactions, healthcare data, and cloud-hosted applications.
Modern WANs generate multivariate telemetry streams---latency, jitter, packet
loss, throughput, congestion---at sub-minute granularity across hundreds of
geographically distributed links.
Despite this data richness, operational monitoring still relies predominantly on
hand-tuned per-device threshold rules (e.g., ``alert if latency exceeds 200\,ms'')
that must be manually maintained, generate frequent false alarms under normal
load spikes, and critically \emph{fail to detect slow, multi-metric degradation
patterns} that precede SLA breaches~\citep{schummer2024}.

The significance of this problem is growing: misconfigured thresholds in
large enterprises can leave genuine WAN degradation undetected for hours,
causing downstream application failures that are expensive to diagnose after
the fact~\citep{chandola2009}.
Supervised machine learning offers a principled alternative by learning
degradation signatures directly from historical telemetry, but systematic
evaluation of which models, features, and evaluation strategies work best in
this setting remains limited in the open literature.

\citet{schummer2024} is a high-impact paper addressing
exactly this gap: it proposes a structured supervised ML framework for WAN
anomaly detection, evaluates Logistic Regression, Decision Tree, Random Forest,
and SVM on real enterprise telemetry, and draws two reproducible, testable
conclusions: (1)~non-linear models outperform a linear baseline, and
(2)~feature engineering quality dominates model complexity.
These claims are concrete, the baseline is accessible, and the methodology
is described in sufficient detail to permit reproduction---making it an ideal
target for a TinyReproductions study.

This project reproduces these claims at small scale using a synthetic but
statistically faithful dataset, and extends the comparison with a PyTorch MLP.
\end{comment}

Enterprise Networks are responsible for transmitting critical, time-sensitive data, 
including financial transactions, healthcare information, and cloud-based applications. 
Modern WANs generate detailed telemetry data, such as latency, jitter, packet loss, 
throughput, and congestion, at minute-level intervals across numerous locations. 
Despite the availability of this data, most monitoring systems continue to rely on manual, 
device-specific threshold rules (for example, “alert if latency exceeds 200 ms”). 
These rules require frequent updates, often produce false alarms during normal fluctuations, 
and typically fail to detect gradual, multi-metric issues that precede service-level 
agreement (SLA) breaches~\citep{schummer2024}.

This challenge is increasingly significant: in large enterprises, poorly configured thresholds 
may allow genuine WAN problems to persist undetected for extended periods, resulting in 
application failures that are difficult and costly to resolve~\citep{chandola2009}. 
Supervised machine learning offers potential by identifying degradation patterns from historical 
data; however, limited research exists regarding the optimal models, features, and evaluation 
methods for this application. 

Schummer et al. (2024) address this gap by introducing a supervised machine learning 
framework for WAN anomaly detection, evaluating Logistic Regression, Decision Tree, 
Random Forest, and Support Vector Machine (SVM) models on real enterprise data. The authors 
present two specific, testable claims: (1) non-linear models outperform linear models, 
and (2) feature engineering has a greater impact than model complexity. The study provides 
a clear baseline and reproducible methodology, making it suitable for a TinyReproductions 
study. The present project replicates these claims on a small scale using a synthetic yet 
realistic dataset and extends the comparison to include a PyTorch-based multilayer perceptron (MLP).

Because the original enterprise WAN telemetry dataset used in the reference study is proprietary 
and not publicly available, this project utilizes a synthetic but statistically realistic 
multivariate dataset. The dataset is constructed to replicate key statistical properties reported 
in the original work, including anomaly rate, temporal correlations, and degradation patterns in 
latency, packet loss, and jitter. This allows the study to approximate the original experimental 
conditions while ensuring reproducibility and transparency.

\paragraph{Contributions beyond the original paper.}
\begin{enumerate}
  \item A self-contained four-step Python pipeline that reproduces the paper's
        key experimental trends in under 10 minutes on a single CPU.
  \item A device-aware Tukey-\iqr labeling strategy that mirrors the paper's
        unsupervised ground-truth generation without requiring proprietary data.
  \item An extended model comparison that adds a PyTorch MLP, showing all four
        classifiers achieve near-identical performance ($\leq$0.013 \prauc spread),
        with Logistic Regression achieving the highest \prauc (0.870).
  \item Quantitative evidence that the non-linearity advantage claimed in the original
        paper does not emerge on synthetic telemetry: LR matches RF ($\Delta$\prauc $=
        0.002$), suggesting the advantage is dataset-dependent.
\end{enumerate}

\section{Related Work}
\label{sec:related}
% ─────────────────────────────────────────────────────────────────────────────

\paragraph{Anomaly detection surveys.}
\citet{chandola2009} provide the canonical taxonomy of anomaly
detection, distinguishing point, contextual, and collective anomalies.
WAN SLA degradation falls primarily in the collective category: no single
measurement is extreme in isolation, but a sustained pattern across multiple
metrics constitutes an anomaly.
Classical statistical approaches (e.g., CUSUM, exponential smoothing) struggle
with this setting because they operate metric-by-metric and lack cross-metric
correlation modeling.
Our reproduction explicitly addresses this with multi-metric feature engineering
and multi-metric labeling (any flagged metric triggers the anomaly label).

\paragraph{ML-based network anomaly detection.}
A growing body of work applies supervised and unsupervised ML to network traffic
anomaly detection~\citep{schummer2024}.
Existing studies often evaluate on balanced or near-balanced datasets, use
ROC-AUC as the primary metric, and do not account for the severe class imbalance
($\sim$10--20\% anomaly rates) typical of operational WAN telemetry.
We depart from this convention by using \prauc as the primary metric and by
explicitly weighting loss functions and class priors for all four models, which
we argue is the correct evaluation protocol for imbalanced anomaly detection.

\paragraph{Temporal feature engineering.}
Rolling-window statistics capture sustained deviations that instantaneous
measurements miss~\citep{schummer2024}.
Our implementation uses multi-horizon windows (15\,min--24\,h), lag and delta
features for rate-of-change sensitivity, and cyclical time encodings for diurnal
and weekly patterns---all from standard libraries without proprietary tools.

\paragraph{Model interpretability.}
SHAP~\citep{lundberg2017} provides attribution values grounded in cooperative
game theory. \texttt{TreeExplainer} computes exact SHAP values in polynomial
time, making it practical for the feature-rich tabular WAN telemetry setting.
We use SHAP to verify model decisions are driven by meaningful temporal patterns.

\section{Original Paper and Claims to Reproduce}
\label{sec:claims}
% ─────────────────────────────────────────────────────────────────────────────

\citet{schummer2024} evaluate Logistic Regression,
Decision Tree, Random Forest, and SVM on a proprietary enterprise WAN telemetry
dataset.
Their key empirical findings, which we target for reproduction, are:

\begin{enumerate}
  \item \textbf{Non-linear models outperform the linear baseline.}
        Random Forest and SVM achieve higher F1 and AUC than Logistic Regression.
  \item \textbf{Feature engineering quality dominates model complexity.}
        Carefully constructed temporal and statistical features are the primary
        performance driver; swapping classifiers yields smaller gains than
        improving features.
  \item \textbf{Imbalance-aware evaluation is required.}
        Due to rare anomaly rates, models must be evaluated with
        imbalance-sensitive metrics (\prauc rather than accuracy).
\end{enumerate}

Because the original dataset is proprietary, we replace it with a synthetic
dataset whose statistical properties mirror those described in the paper
(see Section~\ref{sec:setup}).

\section{Experimental Setup}
\label{sec:setup}
% ─────────────────────────────────────────────────────────────────────────────

\subsection{Dataset and Labeling}

We generate a synthetic multivariate WAN telemetry dataset spanning 14 days
across 20 simulated network sites, each with three link types (MPLS, Broadband,
LTE/5G), sampled every 15 minutes.
Five metrics are recorded per link: \texttt{latency\_ms}, \texttt{jitter\_ms},
\texttt{loss\_pct}, \texttt{throughput\_mbps}, and \texttt{congestion\_pct}.
The final dataset contains 80{,}640 samples.
Anomalies are injected as gradual multi-metric degradation events with realistic
ramp-up and ramp-down profiles that overlap the tail of the normal distribution,
matching the paper's characterization of slow SLA degradation as the hardest
failure mode for rule-based detectors.

\paragraph{Anomaly labeling.}
Ground-truth labels are produced by a device-aware Tukey-\iqr rule applied
independently within each \texttt{(site\_id, link\_id)} group:
\begin{equation}
  \text{flag}_{m,d} =
    \mathbf{1}\!\left[x_{m,d} < Q_1^{(m,d)} - 1.5\,\mathrm{IQR}^{(m,d)}\right]
    \;\vee\;
    \mathbf{1}\!\left[x_{m,d} > Q_3^{(m,d)} + 1.5\,\mathrm{IQR}^{(m,d)}\right]
\end{equation}
where $m$ indexes the metric and $d$ the device group.
A sample is labeled anomalous if at least one flag fires
(\texttt{min\_breach\_metrics}$=1$).
Per-device grouping is essential: a 200\,ms latency reading is anomalous for
a fiber link but normal for an LTE link---global thresholds produce misleading
labels.
This yields a label distribution of 84.0\% normal / 16.0\% anomalous.

\paragraph{Label noise.}
To simulate imperfect labeling in real deployments, 15\% of anomaly labels are
randomly flipped to normal (anomaly$\to$normal only, preserving the class ratio)
using a fixed random seed.
This decouples the supervised label from the raw IQR breach indicator, forcing
models to learn temporal patterns rather than memorizing the labeling rule.

\subsection{Feature Engineering}

Raw metric values are augmented with four feature categories:
\textbf{(i)}~rolling statistics (mean, std, max) over 15\,min, 60\,min, 6\,h,
and 24\,h windows computed within device groups to prevent cross-device leakage;
\textbf{(ii)}~lag ($x_{t-1}$) and delta ($\Delta x_t = x_t - x_{t-1}$) features
per metric to capture rate-of-change signals;
\textbf{(iii)}~cyclical time encodings (sine/cosine of hour-of-day and day-of-week)
to capture diurnal and weekly traffic patterns without ordinality artifacts; and
\textbf{(iv)}~binary IQR breach indicators (\texttt{flag\_<metric>}) retained as
features alongside raw and rolling signals.
The full feature matrix has $\approx$100 columns.
Missing values at window boundaries are imputed with per-column medians.

\subsection{Train/Test Split and Models}

We use a \textbf{time-aware split}: the earliest 70\% of samples by timestamp
form the training set (56{,}460 samples, cutoff 2025-01-10) and the remaining
30\% form the test set (24{,}180 samples), strictly preventing future information
from appearing in training.
Four classifiers are trained:
\textbf{(i)}~\textit{Logistic Regression (LR)}: \texttt{StandardScaler} $\to$
\texttt{LogisticRegression} (\texttt{class\_weight=balanced},
\texttt{max\_iter=2000}), the linear baseline;
\textbf{(ii)}~\textit{Random Forest (RF)}: 300 trees with
\texttt{class\_weight=balanced\_subsample};
\textbf{(iii)}~\textit{SVM RBF}: \texttt{StandardScaler} $\to$ \texttt{SVC(rbf)},
\texttt{class\_weight=balanced};
\textbf{(iv)}~\textit{PyTorch MLP}: two hidden layers
(input\;$\to$\;64\;$\to$\;32\;$\to$\;1, ReLU, 20\% dropout), trained for 25
epochs using \texttt{BCEWithLogitsLoss} with positive-class weight
$w^+\!=\!n_\mathrm{neg}/n_\mathrm{pos}\!\approx\!5.1$ to address class
imbalance.
\textbf{Evaluation metric:} \prauc is used as the primary metric because
accuracy is misleading at a $\sim$5:1 class ratio (predicting all-normal gives 84.0\%
accuracy while detecting zero anomalies); the random-classifier \prauc baseline
equals the true positive rate, 0.160.

\subsection{Key Findings}

\paragraph{Finding 1 — All models are $\approx$5.4$\times$ above the random baseline.}
Every model achieves \prauc $\approx 0.865$ vs.\ 0.160 for a random classifier
(Table~\ref{tab:results}, Figure~\ref{fig:prauc} left), confirming that
temporal feature engineering carries strong predictive signal regardless of
model choice and validating the core premise of the supervised framework.

\paragraph{Finding 2 — Feature engineering dominates model complexity.}
All four models lie within 0.013 \prauc of each other---from the simplest
linear model (LR: 0.870) to a neural network (MLP: 0.857)---directly
corroborating Schummer et al.'s central claim that feature representation,
not model family, is the binding performance constraint.

\paragraph{Finding 3 — All models are well-calibrated at $\theta=0.5$.}
All models achieve $\geq$99.5\% recall and predict $\approx$18.5\% anomaly rate
against a true rate of 16.0\% (Figure~\ref{fig:prauc} right), a slight
over-prediction consistent with class-balanced training pushing decision
boundaries toward sensitivity.

\paragraph{Finding 4 — Linear model matches non-linear models.}
Logistic Regression achieves the highest \prauc (0.870), marginally outperforming
RF (0.868), SVM (0.865), and MLP (0.857). This partially contradicts the original
paper's non-linearity claim, suggesting the advantage may be dataset-dependent
and require the complexity of real enterprise telemetry to emerge.

\subsection{SHAP Feature Importance}

Figure~\ref{fig:shap} shows SHAP TreeExplainer values for the Random Forest model.
Long-horizon rolling statistics (60\,min and 6\,h means/maxima of latency and
loss metrics) are the top contributors, confirming that \emph{sustained}
degradation patterns---not instantaneous spikes---are the primary discriminative
signal.
This aligns with the paper's description of slow SLA degradation as the hardest
failure mode for threshold-based monitors.
IQR breach indicators (\texttt{flag\_*}) appear in the top 10 features, providing
complementary signal to raw rolling statistics.

Feature importance analysis using SHAP further supports this finding, showing that 
latency, packet loss, and jitter-related features are the primary drivers of anomaly 
detection decisions. In particular, short-term aggregation features and threshold-based 
indicators contribute significantly, reflecting known characteristics of network 
degradation in enterprise WAN environments.

\section{Discussion}
\label{sec:discussion}
% ─────────────────────────────────────────────────────────────────────────────

\subsection{Comparison to Original Paper Results}

The original paper reports Random Forest AUC $> 0.9$ in several scenarios.
Our \prauc $\approx 0.865$ is in a comparable range and reflects a synthetic
dataset with clearly-defined anomaly events; the original paper primarily uses
ROC-AUC, whereas \prauc is the more appropriate imbalance-sensitive metric.

The \emph{relative trends} reproduced faithfully:
\begin{itemize}
  \item All models benefit from temporal feature engineering: \prauc $\approx 5.4\times$
        the random baseline (0.865 vs.\ 0.160).
  \item Feature engineering quality, not model architecture, drives performance
        (all models within 0.013 \prauc).
\end{itemize}

One notable discrepancy: Logistic Regression achieves the \emph{highest} \prauc (0.870),
directly contradicting the original non-linearity claim. With synthetic anomalies that
reliably cross Tukey fences, a linear boundary suffices; the non-linearity advantage
likely requires the overlapping, ambiguous degradation patterns of real telemetry.

\subsection{Limitations}

Three limitations merit future work.
(1)~\texttt{flag\_<metric>} features are derived from the same Tukey-\iqr rule used
to generate labels, creating potential leakage; a full ablation without these features
would quantify their contribution.
(2)~All 20 devices appear in both partitions; a held-out device evaluation would test
generalization to unseen links.
(3)~Synthetic data does not capture the full complexity of real enterprise telemetry,
which likely drives the non-linearity advantage reported in the original paper.

\section{Conclusion}
\label{sec:conclusion}
% ─────────────────────────────────────────────────────────────────────────────
\begin{comment}
This reproduction confirms one central claim of Schummer et al.\ (2024) and
partially challenges another.
(1)~Supervised ML with temporal feature engineering detects WAN anomalies at
$\approx$5.4$\times$ the precision of a random classifier (\prauc 0.865 vs.\ 0.160),
strongly validating that engineered features carry genuine signal.
(2)~Feature engineering quality is the primary performance driver: all four
classifiers cluster within 0.013 \prauc of each other, from LR (0.870) to
MLP (0.857).

The non-linearity claim is \emph{not confirmed}: Logistic Regression achieves the
\emph{highest} \prauc (0.870), suggesting the advantage is dataset-dependent and
requires real-world telemetry complexity to emerge. Future work should ablate
flag-feature leakage, test cross-device generalization, and evaluate on real
telemetry to determine whether the non-linearity benefit is recoverable.
\end{comment}
This project reproduced and extended a machine learning–based anomaly detection 
framework for enterprise WAN telemetry data. Using a synthetic but statistically 
realistic dataset, the study evaluated multiple supervised models, including 
Logistic Regression, Random Forest, Support Vector Machine, and a PyTorch-based 
Multi-Layer Perceptron (MLP), for detecting anomalies in latency, packet loss, 
and jitter metrics.

The results demonstrate that temporal feature engineering plays a dominant role 
in anomaly detection performance, as all evaluated models achieved similar PR-AUC 
values significantly above the random baseline. While the MLP did not outperform 
classical models in overall performance, it exhibited higher recall, highlighting 
an important trade-off between false positives and missed anomalies in practical 
network monitoring scenarios.

Overall, the findings suggest that well-designed feature engineering can be more 
impactful than model complexity for structured telemetry data. Future work will 
focus on improving robustness by removing threshold-derived features, evaluating 
generalization across unseen devices, and refining model outputs for real-world 
deployment.
